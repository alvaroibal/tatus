## Vision

### 10x Check

La versión 10x no añade más features a una app de paternidad. Crea **un artefacto generacional**. El padre no solo usa la app durante el primer año — la app genera el legado. Existe en 2044, cuando el hijo tenga 18 años, sin depender de ninguna empresa, servidor, o formato propietario.

La app une dos cosas que ninguna app existente combina:
1. **Guía semanal de paternidad** (para el padre ahora)
2. **Archivo permanente para el hijo** (para el futuro)

Wonder Weeks es solo la primera mitad. Tinybeans es solo la segunda. Nadie une ambas con privacidad total.

### Platonic Ideal

Cada domingo por la mañana, el padre abre la app. La semana 23 se acaba de desbloquear. Lee qué está experimentando su hijo físicamente (empieza a reconocer su cara), mentalmente (protoconversaciones, turno de escucha), emocionalmente (ansiedad de separación emergente). Lee qué debe trabajar él: presencia física, contacto visual, hablar mientras hace tareas.

Escribe tres líneas en el diario. Anota el peso y la altura de esta semana. Marca la semana como hito "primera vez que reconoció mi cara".

A los 18 años, su hijo recibe una caja. Dentro hay un USB. Lo abre. Es un HTML — sin internet, sin cuenta, sin empresa. Lee 936 cartas de su padre, una por semana de su vida. Ve el timeline de hitos. Ve la curva de su crecimiento. Sabe exactamente en qué pensaba su padre cuando tenía 4 meses, 8 meses, 2 años.

Eso no existe. Eso es el platonic ideal.

### Schema Dexie V1

```typescript
// Dexie version 1 — schema completo V1
const db = new Dexie('tatus-db')
db.version(1).stores({
  profiles: '++id,name',
  diaryEntries: '++id,profileId,date,week,milestone',
  growthRecords: '[profileId+week],date',  // compound primary key: upsert con put({profileId,week,...}) es atómico
  appConfig: 'id'         // id siempre = 1 (singleton), insertar con put({id:1,...})

interface BabyProfile {
  id?: number;
  name: string;
  birthDate: Date;    // fecha de nacimiento real
  createdAt: Date;
}

interface DiaryEntry {
  id?: number;
  profileId: number;  // FK → BabyProfile.id
  date: Date;
  week: number;       // calculado al crear: Math.floor((Date.now() - birthDate.getTime()) / 604800000)
  text: string;
  milestone: boolean;
}

interface GrowthRecord {
  profileId: number;  // parte del compound PK — no hay id auto-increment
  week: number;       // parte del compound PK

  weightKg?: number;
  heightCm?: number;

// Upsert: await db.growthRecords.put({ profileId, week, weightKg, heightCm, date })
// put() con compound PK auto-inserta o actualiza por (profileId+week) sin búsqueda previa.

  activeProfileId: number;
}
```

### Cálculo de semana y timezone

```typescript
// Semana cronológica. Timezone-aware: medianoche local de cada fecha.
// DST fix: calcular días primero con Math.round (absorbe la hora perdida/ganada por DST),
// luego dividir por 7 con Math.floor. Sin el Math.round, el cambio horario de España
// (último domingo de marzo/octubre) puede causar off-by-one en el día exacto del anniversary.
const MS_PER_DAY = 86_400_000
const DAYS_PER_WEEK = 7

function calculateWeek(birthDate: Date): number {
  const birthMidnight = new Date(birthDate)
  birthMidnight.setHours(0, 0, 0, 0)
  const todayMidnight = new Date()
  todayMidnight.setHours(0, 0, 0, 0)
  const days = Math.round((todayMidnight.getTime() - birthMidnight.getTime()) / MS_PER_DAY)
  return Math.floor(days / DAYS_PER_WEEK)
}
// Negativo → before-birth mode. 0 → primera semana.
```

### Profile context — `useCurrentProfile()` hook

```typescript
// src/hooks/useCurrentProfile.ts
// Contexto global que lee AppConfig + BabyProfile UNA VEZ.
// Todos los componentes consumen este hook en lugar de leer AppConfig directamente.
export function useCurrentProfile(): { profile: BabyProfile | null; currentWeek: number } {
  const config = useLiveQuery(() => db.appConfig.get(1))
  const profile = useLiveQuery(
    () => config?.activeProfileId ? db.profiles.get(config.activeProfileId) : undefined,
    [config?.activeProfileId]
  )
  const currentWeek = profile ? calculateWeek(profile.birthDate) : -1
  return { profile: profile ?? null, currentWeek }
}
```


### Before-birth mode

Si `calculateWeek(birthDate) < 0`:
- La app muestra pantalla especial: "Faltan X días para conocerte" (X = Math.abs(days until birth))
- El diario prenatal está disponible (entradas con week = -N)
- La sección de etapas muestra placeholder: "El contenido de desarrollo se activa cuando nazca tu hijo"
- La sección de vacunas muestra fechas estimadas con aviso "(fecha estimada)"
- El StageCard y el Growth tracker NO están disponibles

### Corrección de birthDate post-setup

Disponible en Settings. Al cambiar la birthDate:
1. `profiles.update(activeProfileId, { birthDate: newDate })` 
2. Las diary entries conservan su campo `week` original (no se reescriben)
3. Los growthRecords idem
4. Los unlocks semanales se recalculan desde el nuevo birthDate automáticamente

### Multi-baby onboarding UX

- **Sin perfiles (first install):** onboarding obligatorio (nombre + birthDate) → crea profile, establece activeProfileId=1 en AppConfig
- **Un perfil:** app normal, sin selector visible
- **Dos+ perfiles:** selector de perfil en la esquina superior de Home (avatar inicial del nombre)
- **Borrar el último perfil:** bloqueado — la app no permite quedarse sin perfiles. Mostrar: "No puedes borrar el único perfil."

**Lookup con doble fallback (obligatorio en StageCard):**
```typescript
function getStage(week: number, stages: Stage[]): Stage {
  // 1. Búsqueda exacta por rango
  const exact = stages.find(s => week >= s.weekStart && week <= s.weekEnd)
  if (exact) return exact
  // 2. Floor search: entrada más cercana por debajo (gap mid-array)
  const floor = [...stages].reverse().find(s => s.weekStart <= week)
  if (floor) return floor
  // 3. Última entrada (week > 51 o array vacío)
  return stages[stages.length - 1]
}
// Nunca devuelve undefined — StageCard puede asumir stage siempre definido.
```

### Export/Import JSON envelope

```json
{
  "schemaVersion": 1,
  "exportedAt": "2026-06-13T20:00:00Z",
  "data": {
    "profiles": [...],
    "diaryEntries": [...],
    "growthRecords": [...],
    "appConfig": { "id": 1, "activeProfileId": 1 }
  }
}
```

El import es **replace-all**: antes de escribir, la transacción Dexie borra todas las tablas y luego escribe el backup completo. Los profileIds son coherentes post-import. Si ya hay datos en la DB: mostrar confirm dialog "Este import reemplazará todos tus datos. ¿Continuar?" antes de proceder.

El import valida `schemaVersion === 1` antes de escribir. Si no coincide: mostrar error "Este archivo es de una versión incompatible."

### Unlock semanal — diseño visual

Cuando se desbloquea la semana nueva (cumpleaños semanal a las 00:00 local):
- La tarjeta de la nueva semana estaba grisada con un candado (CSS: `opacity: 0.4, filter: grayscale(1)`)
- Al abrir la app el día del unlock: la tarjeta hace transición suave (`transition: opacity 0.6s ease, filter 0.6s ease`) de grisada a normal
- Un badge "¡Nueva semana!" aparece sobre la tarjeta durante la primera visita del día (se borra en localStorage al ver)
- El localStorage key es: `tatus_last_week_seen_{profileId} = weekNumber`

### Recharts — Growth tracker UX

- Gráfica `LineChart` con dos líneas (peso en kg, altura en cm en eje Y secundario)
- Si < 2 puntos de datos: mostrar mensaje vacío "Añade tu primera medida para ver la gráfica"
- Puntos interactivos: tap en un punto → tooltip con fecha y valores
- Si no hay ninguna medida: la pantalla muestra el formulario de input directamente, sin gráfica

### Timeline de hitos — UX

- Puntos en línea de tiempo: solo entradas con `milestone: true`
- Tap en punto con UNA entrada: abre esa entrada del diario
- Tap en punto con MÚLTIPLES entradas de esa semana: muestra lista de entradas de esa semana para seleccionar

## Temporal Interrogation (decisiones que el implementador necesita saber ANTES)

- **Hora 1 (foundations):** El scaffold de Dexie version(1) está arriba en Architecture Notes — no hay ambigüedad. El Workbox config de vite-plugin-pwa: estrategia `generateSW` con `workbox: { globPatterns: ['**/*.{js,css,html,ico,png,svg,json}'] }` para precachar todo el bundle incluyendo el JSON de contenido.
- **Hora 2-3 (core logic):** La función `calculateWeek` está en Architecture Notes. Edge case crítico: si el resultado es negativo, NOT un número entre 0 y infinity — renderizar before-birth mode directamente, no semana "-3".
- **Hora 4-5 (content):** Para stages.json, el lookup es: `stages.find(s => week >= s.weekStart && week <= s.weekEnd)`. Si week > 51 (un año+): devolver la última entrada del array. El contenido de semanas 52+ puede usar el mismo de la semana 51 hasta que se extienda el JSON.
- **Hora 6+ (polish):** El unlock badge se guarda en `localStorage.setItem('tatus_last_week_seen_1', weekNumber)`. Al montar el componente: si `currentWeek > storedWeek`, mostrar badge. En unmount o al tap: actualizar stored. Esto NO requiere state global — es puro localStorage.

## Build Version Tracking

En `vite.config.ts`, exponer el git hash como variable de entorno:
```typescript
// vite.config.ts
import { execSync } from 'child_process'

// IMPORTANTE: execSync lanza si no hay repo git o no hay commits todavía.
// Fallback 'dev' para el primer build (antes del primer commit).
let gitHash = 'dev'
try {
  gitHash = execSync('git rev-parse --short HEAD').toString().trim()
} catch { /* no git repo or no commits yet */ }

export default defineConfig({
  base: '/tatus/',   // REQUERIDO para GitHub Pages — sin esto todos los assets dan 404
  define: { __APP_VERSION__: JSON.stringify(gitHash) }
})
```
En Settings footer: `Tatuś — build {__APP_VERSION__}`. Permite saber exactamente qué versión está deployada sin abrir la terminal.

## Error Boundary + iOS Private Browsing Guard

```typescript
// src/main.tsx
import { createRoot } from 'react-dom/client'
import App from './App'
import { ErrorBoundary } from './components/ErrorBoundary'

// Guard: iOS Safari en modo privado bloquea IndexedDB con SecurityError.
// Un PWA instalado (Add to Home Screen) NUNCA tiene este problema.
// Este guard solo aplica si alguien abre la URL en Safari privado antes de instalar.
async function idbAvailable(): Promise<boolean> {
  try {
    const req = indexedDB.open('tatus-idb-test')
    await new Promise<void>((res, rej) => {
      req.onerror = () => rej(req.error)
      req.onsuccess = () => { req.result.close(); indexedDB.deleteDatabase('tatus-idb-test'); res() }
    })
    return true
  } catch {
    return false
  }
}

const available = await idbAvailable()
if (!available) {
  document.getElementById('root')!.innerHTML =
    '<div style="padding:2rem;font-family:system-ui">Tatuś necesita almacenamiento local. ' +
    'Si estás en modo privado, ábrelo en Safari normal e instálalo desde ahí.</div>'
} else {
  createRoot(document.getElementById('root')!).render(
    <ErrorBoundary
      fallback={<div>Algo salió mal al cargar los datos. <button onClick={() => window.location.reload()}>Reintentar</button></div>}
    >
      <App />
    </ErrorBoundary>
  )
}
```
